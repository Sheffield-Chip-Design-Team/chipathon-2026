# SPI Master (→ SX1257)

Control block. See [System Architecture](../System%20Diagram.md) for context.

**Owner:** TBD
**Status:** Not started

## Function

SPI master driven by PicoRV32 via Wishbone. Writes configuration registers to all four SX1257s — frequency, gain, and mode settings. One shared MOSI/MISO/SCK bus with four separate chip selects (`SX1257_CS[3:0]`).

## Protocol

SX1257 SPI: Mode 0, MSB first, up to 10 MHz. 16-bit transactions: `[15] R/W̄, [14:8] address, [7:0] data`.

Firmware writes:
```c
// Write to SX1257_n register addr with value data:
spi_master_write(n, addr, data);
// Expands to: assert CS[n], shift 0b0_aaaaaaa_dddddddd, deassert CS[n]
```

SX1257 register writes used during operation:
- `RegMode` (0x00) — RX/TX mode switching (e.g. `0x03` = RX, `0x0D` = TX+PA, `0x01` = standby)
- `RegFrfRxMsb/Mid/Lsb` (0x01–0x03) — RX PLL frequency
- `RegFrfTxMsb/Mid/Lsb` (0x04–0x06) — TX PLL frequency
- `RegRxAnaGain` (0x0C) — AGC gain setting
- `RegTxGain` (0x08) — TX DAC and mixer gain

## Pass-through mode

When the host writes `SX_CTRL[1]=1` (START), the SPI master issues a single 16-bit SX1257 transaction using `SX_TARGET`, `SX_ADDR`, `SX_DATA`, and `SX_CTRL[0]` (RNW) from the register bank. `BUSY` is asserted for the duration; `SX_DATA` is overwritten with the MISO byte on a read. This path shares the same MOSI/MISO/SCK/CS signals as firmware-initiated transactions — firmware must not assert any `SX1257_CSn` while `BUSY` is high.

See [Register Map](../Register%20Map.md) `0xCA`–`0xCD` for the full protocol and typical init register list.

## Implementation notes

**SCK divider.** Divide 32 MHz by 4 (minimum) = 8 MHz SCK. Expose as a parameter; 10 MHz SX1257 limit leaves margin.

**Shared bus / bus conflict.** `SPI_MOSI`, `SPI_MISO`, `SPI_SCK` are shared with the SPI slave (host interface). The SPI master drives MOSI/SCK only when a `SX1257_CSn` is asserted. The SPI slave tristates MISO when `HOST_CS` is not asserted. No additional arbitration needed if firmware never initiates an SX1257 transaction during a host SPI access.

**MISO capture.** Latch `SPI_MISO` on the rising edge of SCK. SX1257 read responses used only for diagnostic register verification — not in the normal operating path.

**Clock domain.** This block is fully synchronous — SCK is generated by dividing `clk_32m`, so MOSI, SCK, and CS are all in the 32 MHz domain. No CDC required on the SPI master itself. However, if any SX1257 DIO output pins (`pll_lock_rx`, `pll_lock_tx`) are routed back as pad inputs, each requires a 2-FF synchroniser before use in the 32 MHz domain. Alternatively, poll PLL lock by reading `RegModeStatus` (0x11) over this SPI bus — in that case no DIO pads or synchronisers are needed.

**Wishbone interface.** Simple register: write `{chip_sel[1:0], spi_addr[6:0], spi_data[7:0]}` to trigger a transaction. Poll `busy` or wait for Wishbone ack.

## Verification

| Test | Method | Pass criterion |
| --- | --- | --- |
| Single register write | cocotb: trigger write, capture MOSI/SCK/CS | Correct 16-bit sequence, correct CS asserted |
| All 4 chip selects | Write to each SX1257 | Only target CS asserted; others inactive |
| Back-to-back writes | Two writes with no gap | Second write starts after first CS deasserts |
| MISO capture | Drive known value on MISO | Readback data latched correctly |
| Busy flag | Poll during transaction | `busy` high during SCK cycles; low after CS deassert |
